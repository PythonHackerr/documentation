Generate an event

If you haven't done so already, sign up for a free Radar account to get access to the Radar Dashboard and your API keys. You'll also need a geofence to start generating events, so make sure to create a geofence.

How to generate events#

The quickest way to generate location updates to test and visualize entry and exit events is with the Radar Simulator, though you can also generate events through the Toolkit app, the API, or by installing the SDK in your own application.

Navigate to Simulate from the Radar Dashboard.

The simulator is scoped to the environment you've selected in the left navigation (Live or Test). To avoid any production side effects, use the Test environment with the default user provided (simulated-user-id).

Start by clicking anywhere on the map to generate locations and events. Clicking within the boundary of a geofence will generate a geofence entry event. The activity feed is populated in real-time to mirror how our APIs would surface locations and events in your app.

The data generated from the simulator flows through to your enabled integrations in the corresponding environment. The events enabled for your project on the Settings page determine which events are generated and suppressed.

For extra credit, download the Radar Toolkit iOS or Android app to generate events in real life right from your own phone!

How to view, analyze, and debug events#

Once you've started generating events in the simulator, you will see them on the Events page. Select View from an event in the simulator or navigate to Events from the left navigation. You will see the list of events generated from the simulator or Toolkit app.

You can also analyze events over time by selecting the view type menu, Recent, and changing to Analysis.

If you need to debug a specific user, you can also view their detailed activity by navigating to Users, selecting the user of interest, and clicking View user activity. Similar to the simulator, user activity allows you interactively traverse user locations and events for a given time window.

To learn more about geofences and their events, reference the geofences documentation.
